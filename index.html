Synopsis

The main goal of this project is to make blender fast when it comes to exporting or importing 3d objects into the viewport making C modules for python. At present, it consumes more time for large objects such as models which consists of millions of vertices, the python importers and exporters can take a long time when compared to the C module for the same because python is an interpreted language which results in more CPU instructions.This project mainly focuses on STL and PLY formats primarily, which motivates to convert other format's converters modules to C in future.

Benefits

At this stage, blender takes a lot of time to export or import huge models due to an interpreted language(python modules). The C module will speedup import and export time, which makes the user experience smooth and fast. It enables users to export or import more objects without any huge delay, also the baseline model systems can easily handle this in less time comparing with python modules.

Deliverables

⁍ Not only working importers and exporters of STL and PLY but fast.

⁍ Also documentation of the performance of these features.

⁍ No additional documentation needed for users as these are present already.

Project Details

First, need to write some modules for python in C to read or write STL(Standard Tessellation Language) and PLY(Polygon File Format) file formats. Starting with PLY format makes this project easy as there are some libraries in C available but the tricky part is to convert them into a module for python using Python API. After that happens,STL can be done by following similar steps as PLY but making some changes along the path. The final part is importing these modules into python and testing.
